### Task 3: savings telemetry

**Files:** New `src/telemetry.rs` (+ `pub mod telemetry;`), modify `src/proxy.rs` (counters in AppState, tail inspector around the two compressing endpoints' response streams, /health totals), `src/main.rs` (final flush after `axum::serve` returns on the graceful-shutdown path), `src/cache.rs` meta usage, `tests/proxy_test.rs`.

**Flush mechanism (nailed down so the restart test can actually pass):** totals are flushed to the cache meta table (a) on every `/health` read, (b) every 32 counted requests, and (c) in `main.rs` after `axum::serve` returns (graceful ctrl-c). To enable (c), add `pub fn create_app_with_state(cfg, source) -> (Router, Arc<AppState>)` and make `create_app` a thin wrapper — main uses the former. The `totals_survive_restart` integration test relies on (a): it reads `/health` on proxy A (which flushes), then spawns proxy B on the same tempdir path.

Steps:
- [ ] Failing unit tests in `src/telemetry.rs`: `TailBuffer` keeps only last 16 KB across many pushes; `extract_cached_tokens` finds `"cached_tokens":123` in (a) a plain JSON body tail, (b) a final SSE `data:` chunk, (c) split across two pushes straddling the ring boundary (push byte-by-byte in the test); returns None when absent.
- [ ] Failing integration tests (`tests/proxy_test.rs`): `health_totals_accumulate` — two chat requests through wiremock upstream whose response includes `"usage":{"prompt_tokens":100,"prompt_tokens_details":{"cached_tokens":40}}` → /health totals: requests 2, saved_tokens > 0, upstream_cached_tokens 80, responses_with_cache_info 2; `totals_survive_restart` — spawn proxy A with tempdir cache path, send request, read totals; spawn proxy B same path → totals carried over (requests ≥ 1 before new traffic); existing SSE byte-identity + causality tests must still pass UNCHANGED (they are the inspector's no-distortion guard).
- [ ] Implement: `Totals` (AtomicU64 fields + `since`), stored in `Arc` in AppState; increment in chat/completions handlers only when stats were produced; `TailBuffer` (fixed 16 KB ring) + `extract_cached_tokens(tail: &[u8]) -> Option<u64>` via whitespace-tolerant regexes matching BOTH `"cached_tokens"\s*:\s*(\d+)` (OpenAI) and `"cache_read_input_tokens"\s*:\s*(\d+)` (Anthropic/Z.ai — spec §4a; add a unit test for this shape) on the lossy-utf8 tail; wrap ONLY chat/completions response streams in an inspector stream that pushes chunks to the ring and on stream end runs extraction + counter updates (catch-all untouched); /health `totals` object per spec; persistence per the flush mechanism above (load at create_app; flush on /health, every 32 requests, and post-serve in main).
